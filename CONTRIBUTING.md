# 贡献指南

感谢您对本书的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果您发现翻译错误或技术问题，请提交Issue：

1. 描述问题所在章节和位置
2. 说明问题类型（翻译错误、技术错误、格式问题等）
3. 提供建议的修正（可选）

### 提交改进

1. Fork本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的修改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开Pull Request

### 贡献类型

- **翻译改进**：修正翻译错误、优化表达
- **技术更新**：更新过时的技术信息
- **代码示例**：添加更多Linux内核代码示例
- **格式优化**：改进排版和可读性
- **图片补充**：添加更多示意图

## 代码规范

- 使用Markdown格式
- 代码块使用正确的语法高亮
- 保持术语一致性
- 添加适当的注释

## 行为准则

- 尊重他人
- 接受建设性批评
- 关注对社区最有利的事情

## 联系方式

如有问题，请通过GitHub Issue联系我们。
