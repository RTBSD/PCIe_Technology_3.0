# 第11章 物理层 - 逻辑（Gen1和Gen2）

## 上一章回顾

上一章描述了 Ack/Nak 协议：一种自动的、基于硬件的机制，用于确保 TLP（事务层包）在链路上的可靠传输。Ack DLLP（数据链路层包）确认 TLP 的正确接收，而 Nak DLLP 则表示传输错误。该章节描述了正常操作规则以及与 TLP 和 DLLP 错误相关的错误恢复机制。

## 本章内容

本章描述物理层的逻辑子模块。该模块为串行传输准备数据包并恢复数据。完成这一任务需要多个步骤，本章将详细描述这些步骤。本章涵盖与 Gen1 和 Gen2 协议相关的逻辑，这些协议使用 8b/10b 编码。Gen3 的逻辑不使用 8b/10b 编码，将在名为"物理层 - 逻辑（Gen3）"的章节中单独描述。

## 下一章预告

下一章将描述第三代 PCIe（Gen3）的物理层特性。主要变化包括无需将频率加倍即可将带宽相对于 Gen2 加倍的能力，这是通过消除对 8b/10b 编码的需求实现的。在 Gen3 速度下，需要更稳健的信号补偿。实现这些变化比预期的要复杂得多。

## 物理层概述

本物理层概述介绍了 Gen1、Gen2 和 Gen3 实现之间的关系。此后，重点将放在与 Gen1 和 Gen2 相关的逻辑物理层实现上。Gen3 的逻辑物理层实现将在下一章中描述。

物理层位于外部物理链路与数据链路层之间接口的底部。它将来自数据链路层的出站数据包转换为串行比特流，并时钟同步到链路的所有通道（Lane）上。该层还从链路的所有通道恢复比特流。接收逻辑将比特反序列化为符号流，重新组装数据包，并将 TLP 和 DLLP 转发给数据链路层。

```
PCIe 端口分层结构

软件层 - 发送和接收地址及事务信息

    软件层
       ↕
    事务层 (Transaction Layer) - TLP 头部/数据有效载荷/ECRC
       ↕
    数据链路层 (Data Link Layer) - 流控/虚拟通道管理/排序/链路包序列号/LCRC/DLLPs/Ack/Nak CRC
       ↕
    物理层 (Physical Layer) - 物理包起始/编码/并串转换/差分驱动器/链路训练
       ↕
    端口链路 (Port Link)
```

各层的内容是概念性的，并不定义精确的逻辑块，但设计师如果将设计按层划分以匹配规范，他们的实现将受益，因为不断增长的速率对物理层的影响大于其他层。按层职责划分设计允许物理层适应更高的时钟速率，同时尽可能少地改变其他层。

PCIe 规范 3.0 版不使用特定术语来区分由规范版本定义的不同传输速率。考虑到这一点，本书定义并使用以下术语：

- **Gen1** - 第一代 PCIe（修订版 1.x），工作速率为 2.5 GT/s
- **Gen2** - 第二代（修订版 2.x），工作速率为 5.0 GT/s
- **Gen3** - 第三代（修订版 3.x），工作速率为 8.0 GT/s

物理层由两个子模块组成：逻辑部分和电气部分。两者都包含独立的发送和接收逻辑，允许双工通信。

```
物理层结构

        物理层
    ┌─────────────┐
    │    逻辑     │ ←→ Tx/Rx
    │  (Logical)  │
    └─────────────┘
    ┌─────────────┐
    │    电气     │ ←→ Tx+/Tx-/Rx+/Rx-
    │ (Electrical)│
    └─────────────┘
           ↕
         链路
```

## Linux内核实现参考

### 物理层寄存器访问

在Linux内核中，PCIe物理层寄存器通过内存映射I/O访问。以Phytium PCIe控制器为例：

```c
// Phytium PCIe EP寄存器访问
#define PHYTIUM_PCIE_FUNC_BASE(fn)  (((fn) << 14) & GENMASK(16, 14))

static inline void phytium_pcie_writel(struct phytium_pcie_ep *priv, 
                                        u8 fn, u32 reg, u32 value)
{
    writel(value, priv->reg_base + PHYTIUM_PCIE_FUNC_BASE(fn) + reg);
}
```

### 链路宽度配置

在Linux内核中，字节分段通常由硬件自动处理，软件通过配置寄存器设置链路宽度。以DesignWare PCIe控制器为例：

```c
// DesignWare PCIe链路宽度配置
#define PCIE_PORT_LINK_CONTROL    0x710
#define PORT_LINK_MODE_MASK       GENMASK(21, 16)
#define PORT_LINK_MODE_1_LANES    PORT_LINK_MODE(0x1)
#define PORT_LINK_MODE_4_LANES    PORT_LINK_MODE(0x7)
#define PORT_LINK_MODE_8_LANES    PORT_LINK_MODE(0xf)

// 配置为x4链路
dw_pcie_writel_dbi(pci, PCIE_PORT_LINK_CONTROL, PORT_LINK_MODE_4_LANES);
```

## 发送逻辑概述

为简单起见，让我们从该层发送侧的高级概述开始。从顶部开始，我们可以看到来自数据链路层的数据包字节首先进入缓冲区。在这里设置缓冲区是有意义的，因为有时需要延迟来自数据链路层的数据包流，以允许有序集（Ordered Set）数据包和其他项目被注入到字节流中。

对于 Gen1 和 Gen2 操作，这些注入的项目是用于标记数据包边界和创建有序集的控制字符和数据字符。为区分这两种类型的字符，添加了一个 D/K# 位（数据或"控制"）。逻辑可以根据字符的来源确定 D/K# 应该取什么值。

Gen3 操作模式不使用控制字符，因此使用数据模式来组成有序集，以标识传输字节是与 TLP/DLLP 相关还是与有序集相关。在每个 128 位（16 字节）数据块的开头插入一个 2 位同步头（Sync Header）。同步头通知接收器接收到的块是数据块（与 TLP 或 DLLP 相关的字节）还是有序集块。由于 Gen3 模式没有控制字符，因此不需要 D/K# 位。

接下来，来自上层的并行数据字节被发送到字节分段（Byte Striping）逻辑，在那里它们被分散或分段到该链路的所有通道上。数据包的每个字节被传输到每个通道，并且每个要发出的数据包都使用所有活动通道。链路的通道都在同时传输，因此字节必须以足够快的速度进入此逻辑以适应这种情况。例如，如果有八个通道，来自上层的八个并行字节可能到达字节分段逻辑，允许数据同时时钟同步到所有通道。

接下来是加扰器（Scrambler），它将伪随机模式异或到出站数据字节上以混合比特。虽然这看起来可能引入问题，但实际上不会，因为加扰模式是可预测的，不是真正随机的，所以接收器可以使用相同的算法轻松恢复原始数据。如果加扰器不同步，接收器将无法理解比特流，因此为了防止这个问题，加扰器会定期重置（Gen1 和 Gen2）。这样，如果加扰器确实彼此不同步，不久之后它们都会重新初始化并恢复同步。对于 Gen1 和 Gen2 模式，每当检测到 COM 字符时就会发生重新初始化。对于 Gen3 模式，每当看到 EIEOS 有序集时就会发生。Gen3 模式使用更复杂的基于 24 位的加扰器，因此有通过 Gen3 加扰器的替代路径。

对于 Gen1 和 Gen2 模式，加扰的 8 位字符随后由 8b/10b 编码器编码以进行传输。回想一下，字符（Character）是 8 位未编码的字节，而符号（Symbol）是 8b/10b 逻辑的 10 位编码输出。8b/10b 编码有几个优点，但它确实增加了开销。

对于 Gen3，显示了绕过编码器的单独路径。换句话说，数据包的加扰字节在没有 8b/10b 编码的情况下传输。同步位生成器在每个数据包的每 16 字节块之前添加一个 2 位同步头。添加的 2 位同步头标识接下来的 16 字节块是数据块还是有序集块。这个每 16 字节（128 位）添加 2 位同步头是 Gen3 的 128b/130b 编码方案的基础。

最后，符号被序列化为比特流并转发到物理层的电气子模块，传输到链路的另一端。

## 接收逻辑概述

接收逻辑的关键元素包括：

1. **时钟和数据恢复（CDR）**：接收器时钟和数据恢复逻辑从输入比特流中的跳变恢复时钟。这个恢复的时钟忠实地再现了用于发送数据的发送器时钟，并用于将输入比特锁存到反序列化缓冲区中。

2. **符号锁定（Symbol Lock）**：CDR 过程的下一步是找到 Gen1/Gen2 符号边界，并将恢复的时钟除以 10，以将 10 位符号锁存到弹性缓冲区（Elastic Buffer）中。对于 Gen3，下一步是获取块锁定（Block Lock），然后将与块中 16 个字节中的每一个相关联的 8 位符号锁存到弹性缓冲区中。

3. **弹性缓冲区控制**：控制弹性缓冲区的逻辑通过根据需要添加或删除 SKP 符号来调整恢复时钟与接收器本地时钟之间的微小时钟变化。

4. **8b/10b 解码器**：使用 8b/10b 解码器，Gen1/Gen2 符号被解码，从而将 10 位符号转换为 8 位字符。

5. **解扰器（Descrambler）**：解扰器应用发送器使用的相同加扰方法来恢复原始数据。

6. **字节反分段（Byte Un-striping）**：最后，来自每个通道的字节被反分段以形成将转发给数据链路层的字节流。只有 TLP 和 DLLP 被加载到接收缓冲区并发送到数据链路层。

## 发送逻辑详情（仅限 Gen1 和 Gen2）

### Tx 缓冲区

从图的顶部开始，缓冲区接受来自数据链路层的 TLP 和 DLLP，以及指定新数据包何时开始的"控制"信息。如前所述，缓冲区允许我们不时地停止字符流，以便插入控制字符和有序集。如果缓冲区变满，还会向数据链路层发送一个"节流"信号以停止字符流。

### 多路复用器和控制逻辑

多路复用器用于将特殊控制（K）字符插入来自缓冲区的数据流中。只有物理层使用 K 控制字符；它们在传输期间插入，在接收器处删除。多路复用器的四个不同输入是：

1. **发送数据缓冲区**：当数据链路层提供数据包时，多路复用器选通字符流。来自缓冲区的所有字符都是 D 字符，因此当 Tx 缓冲区内容被选通时，D/K# 信号被驱动为高。

2. **起始和结束字符**：这些控制字符被添加到每个 TLP 和 DLLP 的开头和结尾，允许接收器轻松检测数据包边界。有两个起始字符：STP 表示 TLP 的开始，SDP 表示 DLLP 的开始。来自数据链路层的指示器以及数据包类型决定了要插入什么类型的帧字符。还有两个结束字符：正常传输的 END Good 字符（END），以及处理某些错误情况的 END Bad 字符（EDB）。起始和结束字符是 K 字符，因此当插入起始和结束字符时，D/K# 信号被驱动为低。

3. **有序集**：如前所述，控制字符仅由物理层使用，高层看不到。跨链路需要进行一些通信来启动和维护链路操作，这是通过交换有序集来完成的。每个有序集都以一个称为逗号（COM）的 K 字符开头，并根据要传递的有序集类型包含其他 K 或 D 字符。有序集始终与四字节边界对齐，并在各种情况下传输，包括：
   - 错误恢复、启动事件（如热复位）或退出低功耗状态。在这些情况下，在链路上交换训练序列 1 和 2（TS1 和 TS2）有序集。
   - 在周期性间隔，多路复用器插入 SKIP 有序集模式以促进接收器中的时钟容差补偿。
   - 当设备希望将其发送器置于电气空闲状态时，它必须通知链路远端的接收器。多路复用器插入电气空闲有序集来完成此操作。
   - 当设备希望将链路电源状态从 L0s 低功耗状态更改为 L0 全开启电源状态时，它向接收器发送一组快速训练序列（FTS）有序集。接收器使用此有序集将其 PLL 重新同步到发送器时钟。

4. **逻辑空闲序列**：当没有准备好传输的数据包且没有要发送的有序集时，链路在逻辑上处于空闲状态。为了保持接收器 PLL 锁定到发送器的频率，发送器必须继续发送某些内容，因此在这种情况下插入逻辑空闲字符。逻辑空闲非常简单，只不过是一串值为 00h 的数据字符。

## 字节分段（用于宽链路）

示例中显示的下一步是字节分段，尽管这仅在端口实现多个通道（称为宽链路）时才需要。分段意味着字符流中的每个连续出站字符被路由到连续的通道上。使用的通道数在链路训练过程中根据共享链路的两个设备支持的内容进行配置。

### x1 字节分段

单通道链路（x1）的情况不太有趣，因为数据包以字节为单位进入物理层并以相同方式发出，但它说明了字符序列的绘制方式。

## 数据包格式规则

### 一般规则

- 每个数据包的总长度（包括起始和结束字符）始终是四个字符的倍数。这是数据长度以双字为单位测量这一事实的自然延伸。
- TLP 以 STP 字符开始，以 END 或 EDB 字符结束。
- DLLP 以 SDP 开始，以 END 字符结束，长度恰好为 8 个字符（SDP + 6 个字符 + END）。
- 在传输逻辑空闲后启动数据包传输时，STP 和 SDP 字符被放置在通道 0 上。在其他情况下，它们可能从可被 4 整除的通道号开始。
- 接收器的物理层可以监视这些规则的违反情况，并可能将其作为接收器错误报告给数据链路层。

### x1 格式示例

在 x1 链路（仅有一个通道可操作的链路）上传输的数据包格式示例显示了一系列数据包，其中穿插着一个 SKIP 有序集。末尾显示逻辑空闲，表示发送器没有更多数据包要发送并使用空闲字符作为填充的情况。

```
时间 →
通道 0: STP - TLP - END - SDP - DLLP - END - STP - TLP - END - COM - SKP - STP - TLP - END - Idle - Idle - Idle
```

### x4 格式规则

- STP 和 SDP 字符始终在通道 0 上发送。
- END 和 EDB 字符始终在通道 3 上发送。
- 当发送有序集（如 SKIP）时，它必须同时出现在所有通道上。
- 当传输逻辑空闲时，它们必须同时在所有通道上发送。
- 任何违反这些规则的情况都可能作为接收器错误报告给数据链路层。

### x4 格式示例

```
通道:    0      1      2      3
        ┌──────┬──────┬──────┬──────┐
时间 →  │ STP  │ TLP  │ TLP  │ TLP  │
        │ LCRC │ LCRC │ LCRC │ END  │
        │ COM  │ SKP  │ SKP  │ SKP  │  ← SKIP 有序集
        │ SDP  │ DLLP │ DLLP │ END  │
        │ Idle │ Idle │ Idle │ Idle │  ← 逻辑空闲
        └──────┴──────┴──────┴──────┘
```

### 大链路宽度数据包格式规则

当数据包通过 x8、x12、x16 或 x32 链路传输时，适用以下规则：

- 在传输逻辑空闲一段时间后开始传输时，STP/SDP 字符始终在通道 0 上发送。之后，当发送背靠背数据包时，它们只能在可被 4 整除的通道号上发送（通道 4、8、12 等）。
- END/EDB 字符在可被 4 整除然后减一的通道号上发送（通道 3、7、11 等）。
- 如果数据包没有结束在链路的最后一个通道上，并且没有更多准备好发送的数据包，则使用 PAD 符号作为剩余通道号的填充。逻辑空闲不能用于此目的，因为它必须同时出现在所有通道上。
- 有序集必须同时在所有通道上发送。
- 类似地，当使用逻辑空闲时，它们必须同时在所有通道上发送。
- 任何违反这些规则的情况都可能作为接收器错误报告给数据链路层。

### x8 数据包格式示例

在 x8 链路上传输的数据包格式示例显示了一个 TLP，后跟一个 SKIP 有序集、一个 DLLP，最后是一个在通道 3 上结束的 TLP。此时，发送器没有更多准备好发送的数据包，但当前数据包没有延伸到包含所有可用通道。人们可能期望额外的通道用逻辑空闲填充，但这里不起作用，因为空闲必须同时出现在所有通道上。因此需要另一个填充字符，规范编写者选择在这里使用 PAD 控制字符。PAD 唯一使用的其他地方是在训练过程中。最后，由于仍然没有更多数据包要发送，在所有通道上发送逻辑空闲。

```
通道:   0      1      2      3      4      5      6      7
       ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
       │ STP  │ TLP  │ TLP  │ TLP  │ TLP  │ TLP  │ TLP  │ TLP  │
       │ LCRC │ LCRC │ LCRC │ LCRC │ LCRC │ LCRC │ LCRC │ END  │
       │ COM  │ SKP  │ SKP  │ SKP  │ SKP  │ SKP  │ SKP  │ SKP  │ ← SKIP 有序集

       └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

## 加扰器（Scrambler）

示例中的下一步是加扰，旨在防止数据流中的重复模式。重复模式在链路上产生"纯音"，意味着由模式引起的产生比通常更多噪声或 EMI 的一致频率。通过将这种能量扩散到更宽的频率范围来减少这个问题是加扰的主要目标。此外，一个通道上的加扰传输还减少了与宽链路上相邻通道的干扰。这种"空间频率去相关"或减少串扰噪声有助于每个通道上的接收器区分所需信号。

为了帮助接收器保持与加扰序列的同步，控制字符不被加扰，因此即使加扰器不同步也能被识别。此外，COM 控制字符（K28.5）的到来会在每次到达时重新初始化链路两端的加扰器，从而重新同步它们。

### 加扰器算法

规范中描述的加扰器由 16 位线性反馈移位寄存器（LFSR）组成，反馈点实现以下多项式：

```
G(x) = X¹⁶ + X⁵ + X⁴ + X³ + 1
```

```
加扰器结构 (16-bit LFSR)

X0 ──→ X1 ──→ X2 ──→ ... ──→ X15 ──→ XOR ──→ 反馈
                                     ↑
                                     └── XOR ←── X3
                                         ↑
                                         └── XOR ←── X4
                                             ↑
                                             └── XOR ←── X5

LFSR 输出: 8 位并行
数据输入: 8 位字符
输出: Scrambled Data = Data XOR LFSR_Output

工作时钟: 8 倍字节时钟频率 (Gen1: 250MHz 字节时钟 → 2GHz LFSR 时钟)
```

LFSR 以馈送数据字节的时钟频率的 8 倍进行时钟同步，其输出被时钟同步到与 8 位数据字符进行异或以形成加扰数据输出的 8 位寄存器中。

### 加扰器实现规则

- 在多通道链路实现中，与每个通道相关联的加扰器必须协同工作，在每个 LFSR 中保持相同的同步值。
- 加扰仅应用于 'D' 字符，即与 TLP 和 DLLP 以及逻辑空闲（00h）字符相关联的字符。但是，TS1 和 TS2 有序集中的 'D' 字符不被加扰。
- 加扰从不应用于 'K' 字符和有序集内的字符，如 TS1、TS2、SKIP、FTS 和电气空闲。这些字符绕过加扰器逻辑。其中一个原因是确保即使加扰器以某种方式失去序列，它们仍然可以被接收器识别。
- 合规模式字符（用于测试）也不被加扰。
- COM 字符是不被加扰的控制字符，用于在发送器和接收器处将 LFSR 重新初始化为 FFFFh。
- 除了 COM 字符外，LFSR 通常每发送一个 D 或 K 字符就串行前进八次，但在与 SKIP 有序集相关联的 SKP 字符上不前移。原因是接收器可能添加或删除 SKP 符号以执行时钟容差补偿。如果它们不被忽略，改变接收器中的字符数量与发送的字符数量相比将导致接收器 LFSR 中的值与发送器 LFSR 值失去同步。

### 禁用加扰

加扰默认启用，但规范允许出于测试和调试目的禁用它。这是因为测试可能需要控制发送的确切比特模式，由于硬件处理加扰，软件没有合理的方式能够强制特定模式。规范没有定义特定的软件机制来指示物理层禁用加扰，因此这必须是设计特定的实现。

如果设备禁用加扰，这通过发送至少两个在控制字段中设置了适当位的 TS1 和 TS2 来传达给相邻设备。作为响应，相邻设备也禁用其加扰。

## 8b/10b 编码

### 概述

PCIe 的前两代使用 8b/10b 编码。每个通道实现一个 8b/10b 编码器，将 8 位字符转换为 10 位符号。这种编码方案由 IBM 于 1984 年获得专利，如今广泛用于许多串行传输，如千兆以太网和光纤通道。

### 动机

编码为串行传输实现了几个理想的目标。其中三个最重要的目标如下：

1. **将时钟嵌入数据中**：编码确保数据流中有足够的边沿以在接收器处恢复时钟，结果是不需要分布式时钟。这避免了并行总线设计的一些限制，如飞行时间和时钟偏斜。它还消除了分发会引起其他问题（如增加的 EMI 和难以布线）的高频时钟的需求。

   作为此过程的示例，数据字节 00h 的编码结果显示这个没有跳变的 8 位字符转换为具有 5 个跳变的 10 位符号。8b/10b 保证有足够的边沿，确保比特流中的"游程长度"（连续 1 或 0 的序列）在任何条件下都不超过 5 个连续比特。

2. **保持直流平衡**：PCIe 使用交流耦合链路，在路径中串联放置电容器以将信号的直流部分与链路的另一端隔离。这允许发送器和接收器使用不同的共模电压，并使电气设计更容易，因为发送器和接收器之间的路径足够长，它们不太可能有完全相同的参考电压。该直流值或共模电压可以在运行时改变，因为信号被驱动时线路会充电。通常，信号变化如此之快以至于没有时间造成问题，但如果信号平均值主要是一个电平或另一个电平，共模值将看起来漂移。这种称为"直流漂移"的漂移电压会降低接收器处的信号完整性。为了补偿，8b/10b 编码器跟踪发送的最后一个符号的"不一致性"。不一致性或不等性简单地指示前一个符号是否有比 0 更多的 1（称为正不一致性），比 1 更多的 0（负不一致性），或 1 和 0 的平衡（中性不一致性）。例如，如果前一个符号具有负不一致性，下一个应该通过使用更多的 1 来平衡。

3. **增强错误检测**：编码方案还有助于检测传输错误。对于 10 位值，1024 个代码是可能的，但要编码的字符只有 256 个唯一代码。为了保持直流平衡，设计对每个字符使用两个代码，并根据发送的最后一个符号的不一致性选择使用哪一个，因此需要 512 个代码。然而，许多中性不一致性编码具有相同的值（D28.5 就是一个例子），因此并非所有 512 个都被使用。结果，超过一半的可能编码未被使用，如果在接收器处看到将被视为非法。如果传输错误确实改变了符号的比特模式，结果很可能是可以立即识别的这些非法模式之一。

8b/10b 编码的主要缺点是其所需的开销。从接收器的角度来看，实际传输性能降低了 20%，因为每个字节发送 10 位，但接收器只恢复 8 个有用位。这是一个不小的代价，但仍被认为是可以接受的，以获得上述优势。

### 10 位符号的属性

如 8b/10b 编码文献中所述，设计严格来说不是 8 位到 10 位。相反，它实际上是 5 到 6 位编码，然后是 3 到 4 位编码。子块是设计内部的，但它们的存在有助于解释合法符号的一些属性，如下所列。不遵循这些属性的符号被视为无效。

- 比特流从不包含超过五个连续的 1 或 0，即使从一个符号的末尾到下一个符号的开头。
- 每个 10 位符号包含：
  - 四个 0 和六个 1（不一定连续），或
  - 六个 0 和四个 1（不一定连续），或
  - 五个 0 和五个 1（不一定连续）。
- 每个 10 位符号被细分为两个子块：第一个是 6 位宽，第二个是 4 位宽。
  - 6 位子块包含不超过四个 1 或四个 0。
  - 4 位子块包含不超过三个 1 或三个 0。

### 字符表示法

8b/10b 使用特殊的表示法简写。以下是获得给定字符简写的步骤：

1. 将字符分成其 3 位和 5 位子块。
2. 调换子块的位置。
3. 为每个子块创建十进制等效值。
4. 字符采用 Dxx.y 形式表示数据字符，或 Kxx.y 表示控制字符。在此表示法中，xx 是 5 位字段的十进制等效值，y 是 3 位字段的十进制等效值。

```
8b/10b 命名法示例

8b 字符: 0110 1010 (6Ah)

步骤 1: 分成子块
  HGF = 011 (3位)
  EDCBA = 01010 (5位)

步骤 2: 调换子块位置
  EDCBA HGF = 01010 011

步骤 3: 转换为十进制
  01010 = 10
  011 = 3

步骤 4: 最终表示
  D10.3 (数据字符)
```

### 不一致性（Disparity）

**定义**：不一致性是指 10 位符号中 1 和 0 数量之间的不平等，用于帮助保持链路上的直流平衡。具有更多 0 的符号被称为具有负（-）不一致性，而具有更多 1 的符号具有正（+）不一致性。当符号具有相等数量的 1 和 0 时，它被称为具有中性不一致性。有趣的是，大多数字符编码为具有 + 或 - 不一致性的符号，但有些只编码为具有中性不一致性的符号。

**CRD（当前运行不一致性）**：CRD 是有关链路上不一致性当前状态的信息。由于它只是一个位，它只能是正或负，并且在发送下一个符号时并不总是改变。为了解其工作原理，请记住下一个解码的符号可以具有负、中性或正不一致性。如果 CRD 为正，具有负不一致性的出站符号会将其变为负，中性不一致性会保持为正，而正不一致性将是错误，因为 CRD 只有一个位，不能变得更正。

发送器和接收器之间 CRD 的初始状态（在传输任何字符之前）可能不匹配，但事实证明这并不重要。当接收器在训练完成后看到第一个符号时，它会检查不一致性错误，如果发现错误，只需更改 CRD。这不会被视为错误，而只是将 CRD 调整为匹配接收器和发送器。之后，只有两个合法的 CRD 情况：如果新符号具有中性不一致性，它可以保持不变，或者如果新符号具有相反的不一致性，它可以翻转到相反的极性。不合法的是新符号的不一致性与 CRD 相同。这样的事件将是不一致性错误，除非在初始调整后发生错误，否则不应发生。

### 编码过程

实现 8b/10b 编码有不同的方法。最简单的方法可能是实现一个包含所有可能输出值的查找表。然而，这个表可能需要相对大量的门。另一种方法是将解码器实现为逻辑块，这通常是首选，因为它通常导致更小、更便宜的解决方案。编码逻辑的具体细节在参考文献中有详细描述，因此我们将在这里关注其工作原理的大局。

编码后，生成的符号被馈送到串行器，该串行器将各个比特时钟同步输出。

## 控制字符

8b/10b 编码为链路管理提供了几个特殊字符。

| 字符名称 | 8b/10b 名称 | 描述 |
|---------|------------|------|
| COM | K28.5 | 任何有序集中的第一个字符。也用于接收器在训练期间实现符号锁定。 |
| PAD | K23.7 | 数据包填充符 |
| SKP | K28.0 | 用于 SKIP 有序集进行时钟容差补偿 |
| STP | K27.7 | TLP 的起始 |
| SDP | K28.2 | DLLP 的起始 |
| END | K29.7 | 无错误数据包的结束 |
| EDB | K30.7 | 错误或"作废"TLP 的结束 |
| FTS | K28.1 | 用于从 L0s 低功耗状态退出到 L0 |
| IDL | K28.3 | 用于将链路置于电气空闲状态 |
| EIE | K28.7 | 电气空闲退出有序集的一部分，在将链路恢复到全功率之前发送，用于高于 2.5 GT/s 的速度 |

- **COM（逗号）**：其主要功能之一是成为物理层通信中称为有序集的第一个符号。它具有一个有趣的特性，使其两个符号编码都易于在接收器处识别：它们以两个相同极性的比特开始，后跟五个相反极性的比特（0011111010 或 1100000101）。这个特性对于初始训练特别有帮助，当接收器试图理解传入的符号串时，因为它帮助接收器锁定到传入的符号流。

- **PAD**：在多通道链路上，如果要发送的数据包未覆盖所有可用通道且没有更多准备好发送的数据包，则使用 PAD 字符填充剩余通道。

- **SKP（跳过）**：这是作为 SKIP 有序集的一部分使用的，定期发送以促进时钟容差补偿。

- **STP（TLP 起始）**：插入以标识 TLP 的开始。

- **SDP（DLLP 起始）**：插入以标识 DLLP 的开始。

- **END**：附加以标识无错误 TLP 或 DLLP 的结束。

- **EDB（错误结束）**：插入以标识转发设备（如交换机）希望"作废"的 TLP 的结束。这种情况可能在使用"直通模式"的交换机将数据包从入口端口转发到出口端口而不先缓冲整个数据包时产生。在转发过程中检测到的任何错误都会创建一个问题，因为在可以检查数据包是否有错误之前，部分数据包已经在交付中。为了处理这种情况，交换机必须取消已经在路由到目的地的数据包。这是通过作废它来完成的：用 EDB 结束数据包并反转 LCRC。当接收器看到作废的数据包时，它丢弃数据包并且不返回 ACK 或 NAK。

- **FTS（快速训练序列）**：FTS 有序集的一部分，由设备发送以将链路从 L0s 待机状态恢复到全开启 L0 状态。

- **IDL（空闲）**：电气空闲有序集的一部分，发送以通知接收器链路正在过渡到较低功耗状态。

- **EIE（电气空闲退出）**：在 PCIe 2.0 规范中添加，用于帮助电气空闲链路开始唤醒过程。

## 有序集（Ordered Sets）

### 概述

有序集用于链路伙伴的物理层之间的通信，可以被视为通道管理数据包。根据定义，它们是不是 TLP 或 DLLP 的一系列字符。对于 Gen1 和 Gen2，它们始终以 COM 字符开头。有序集同时在所有通道上复制，因为每个通道技术上是一个独立的串行路径。这也允许接收器验证对齐和去偏斜。有序集用于链路训练、时钟容差补偿和改变链路电源状态等。

### TS1 和 TS2 有序集（TS1OS/TS2OS）

训练序列一和二用于链路初始化和训练。它们允许链路伙伴实现位锁定和符号锁定，协商链路速度，并报告与链路操作相关的其他变量。

### 电气空闲有序集（EIOS）

希望进入较低功耗链路状态的发送器在停止传输之前发送此信号。收到后，接收器知道准备进入低功耗状态。EIOS 由四个符号组成：COM 符号后跟三个 IDL 符号。接收器检测此有序集并通过忽略输入错误直到退出电气空闲来准备进入电气空闲。发送 EIOS 后不久，发送器将其差分电压降低到小于 20mV 峰值。

### FTS 有序集（FTSOS）

发送器发送适当数量的这些（最小数量由链路邻居在训练期间给出）以将链路从低功耗 L0s 状态恢复到全操作 L0 状态。接收器检测 FTS，识别链路正在退出电气空闲，并使用它们恢复位和符号锁定。FTS 有序集由四个符号组成：COM 符号后跟三个 FTS 符号。

### SKIP 有序集（SOS）

这由四个符号组成：COM 符号后跟三个 SKP 符号。它定期传输，用于时钟容差补偿。基本上，接收器评估 SOS 并根据需要内部添加或删除 SKP 符号，以防止其弹性缓冲区下溢或溢出。

### 电气空闲退出有序集（EIEOS）

在 PCIe 2.0 规范中添加，定义此有序集以提供退出电气空闲链路状态所需的较低频率序列。对于 8b/10b 编码，EIEOS 使用重复的 K28.7 控制字符作为重复的 5 个 1 后跟 5 个 0 的字符串出现。这种低频字符串产生低频信号，允许更高的信号电压，更容易被接收器检测到。事实上，规范指出这种模式保证接收器将正确检测从电气空闲的退出，这是加扰数据无法做到的。

## 串行器（Serializer）

每个通道上的 8b/10b 编码器馈送一个串行器，该串行器按位顺序将符号时钟同步输出，最低有效位（a）首先移出，最高有效位（j）最后移出。对于每个通道，符号将以 250MHz 或 500MHz 的频率提供给串行器，以支持比该频率快 10 倍的串行比特率，即 2.5 GHz 或 5.0 GHz。

## 差分驱动器

实际将比特流发送到线路上的差分驱动器使用 NRZ 编码。NRZ 简单地意味着不使用特殊或中间电压电平。差分信号改善信号完整性，并允许更高的频率和更低的电压。

## 发送时钟（Tx Clock）

每个通道上的串行化输出由 Tx 时钟信号时钟同步输出。如前所述，时钟频率必须在中心频率的 +/-300ppm 范围内准确（总变化为 600ppm）。关于此时钟源有两个选项。它可以内部生成，也可以从外部参考派生，该参考可能可选可用。PCIe 外围卡规范包括为系统板提供的 100MHz 参考时钟的定义，用于此目的。此参考时钟在内部乘以派生驱动内部逻辑和串行器使用的 Tx 时钟的本地时钟。

## 其他发送主题

### 逻辑空闲

为了保持接收器 PLL 不漂移，在没有 TLP、DLLP 或有序集要传输的时间段内必须传输某些内容，在这些时间段内将逻辑空闲字符注入字符流中。逻辑空闲字符的一些属性：

- 它是一个值为 00h 的 8 位数据字符。
- 发送时，它同时出现在所有通道上，链路被称为处于逻辑空闲状态（不要与电气空闲混淆——输出驱动器完全停止传输且接收器 PLL 失去同步的状态）。
- 逻辑空闲字符被加扰，但接收器可以将其与其他数据区分开来，因为它出现在数据包帧上下文之外（即在 END 或 EDB 之后，但在 STP 或 SDP 之前）。
- 它经过 8b/10b 编码。
- 在逻辑空闲传输期间，仍定期发送 SKIP 有序集。

### 发送信号偏斜

可以理解，发送器应在通道之间引入最小偏斜，以便为路由和其他变化留下尽可能多的接收偏斜预算。规范将发送偏斜值列为 Gen1 500ps + 2 UI，Gen2 500ps + 4 UI，Gen3 500ps + 6 UI。回想一下，UI（单位间隔）表示链路上的一个比特时间，这计算如下：

| 规范版本 | 允许的发送偏斜 |
|---------|--------------|
| Gen1 | 1300 ps |
| Gen2 | 1300 ps |
| Gen3 | 1250 ps |

## 时钟补偿

### 背景

像 PCIe 这样的高速串行传输有一个特定的时钟问题需要解决。接收器从传入的比特流中恢复时钟，并使用它来锁存数据比特，但这个恢复的时钟与接收器的内部时钟不同步，在某些时候它必须开始用其自己的内部时钟为数据时钟同步。即使它们有可选的公共外部参考时钟，它们最多只能在所需频率的指定容差范围内生成内部时钟。因此，其中一个时钟几乎总是比另一个稍快。如果发送器时钟更快，数据包到达的速度将比接收速度快。为了补偿，发送器必须在比特流中注入一些"可丢弃字符"，如果证明有必要避免缓冲区溢出条件，接收器可以丢弃这些字符。对于 PCIe，这些可以删除的字符采用 SKIP 有序集的形式，由 COM 字符后跟三个 SKP 字符组成。

### SKIP 有序集插入规则

发送器需要定期发送 SKIP 有序集，适用以下规则：

- SKIP 有序集必须在 1180 到 1538 个符号时间之间调度插入（符号时间是发送一个符号所需的时间，为 10 比特时间，因此在 2.5 GT/s 时，符号时间为 4ns，在 5.0 GT/s 时，为 2ns）。
- 它们仅在数据包边界插入（不允许中断数据包），必须同时在所有通道上。如果数据包已经在进行中，SKIP 有序集将不得不等待。最大可能的数据包大小需要超过 4096 个符号时间，在此期间应该已经发送了几个 SKIP 有序集。这种情况通过累积应该发出的 SKIP 并在下一个数据包边界注入它们来处理。
- 由于此有序集必须同时在所有通道上传输，多通道链路可能需要在某些通道上添加 PAD 字符，以允许有序集同时在所有通道上传输。
- 在低功耗链路状态期间，用于调度 SKIP 有序集的任何计数器必须复位。当发送器不发送信号时不需要它们，唤醒链路来发送它们是没有意义的。
- 合规模式进行时不得传输 SKIP 有序集。

## 接收逻辑详情（仅限 Gen1 和 Gen2）

### 差分接收器

接收器逻辑的第一部分包括每个通道的差分输入缓冲器。缓冲器感应峰峰值电压差，并确定该差是否代表逻辑 1 或 0。

### 接收时钟恢复

#### 概述

接下来，接收器从输入数据流中的数据比特跳变生成接收时钟（Rx Clock），可能使用 PLL。这个恢复的时钟具有与用于将比特流时钟同步到线路上的发送时钟（Tx Clock）相同的频率（2.5 或 5.0 GHz）。接收时钟用于将入站比特流时钟同步到反序列化器中。反序列化器必须对齐到 10 位符号边界（称为实现符号锁定的过程），然后其符号流输出使用时钟除以 10 的接收时钟版本时钟同步到弹性缓冲区中。即使两者都必须在中心频率的 +/-300ppm 范围内准确，接收时钟可能与本地时钟略有不同，如果是这样，需要补偿。

#### 实现位锁定

回想一下，8b/10b 编码方案保证入站串行符号流包含频繁的跳变。接收器 PLL 使用这些跳变创建与用于将比特流时钟同步出发送器的发送时钟同步的接收时钟。当接收器锁定到发送时钟频率时，接收器被称为已实现"位锁定"。

在链路训练期间，发送器向接收器发送一长串 TS1 和 TS2 有序集，接收器然后使用其中的比特跳变来实现位锁定。在正常操作期间，链路上有足够的跳变，供接收器此后保持位锁定。

#### 失去位锁定

如果链路被置于低功耗状态（如 L0s 或 L1），数据包传输停止，接收器将失去同步。为了避免错误电路将此视为错误，发送器在进入较低功耗状态之前发送电气空闲有序集（EIOS），告诉接收器取消其输入。

#### 重新获得位锁定

当发送器准备将链路从 L0s 状态唤醒时，它发送特定数量的 FTS 有序集（实际数量是设计特定的），接收器使用这些重新获得位和符号锁定。恢复只需要相对较少数量的 FTS，因此恢复延迟很短。因为链路处于 L0s 状态的时间很短，接收器 PLL 通常在开始接收 FTS 之前不会偏离发送时钟太远。如果链路处于 L1 低功耗状态，发送器开始传输 TS1OS。这导致链路重新训练，唤醒时间比 L0s 唤醒时间长。如果链路有更严重的错误，并且在重试 TLP 四次后 Ack/Nak 机制在错误恢复方面不成功，数据链路层向物理层发出信号以重新训练链路。这里同样，在重新训练过程中重新建立位锁定。

### 反序列化器

#### 概述

入站数据被每个通道的反序列化器（串并转换器）用时钟同步到接收时钟中。生成的 10 位符号使用时钟除以 10 的接收时钟版本时钟同步到弹性缓冲区中。

#### 实现符号锁定

当接收逻辑开始接收比特流时，它是 JABOB（只是一堆比特），没有标记来区分符号或任何边界。接收逻辑必须有一种方法来找到 10 位符号的开始和结束，逗号（COM）符号就是为此目的服务的。

COM 符号的 10 位编码包含两个相同极性的比特，后跟五个相反极性的比特（0011111010b 或 1100000101b），使其易于检测。回想一下，COM 控制字符与所有其他控制字符一样，也不会被发送器加扰，这确保了将在接收器处看到所需序列。检测到 COM 后，逻辑知道接收到的下一个比特将是下一个 10 位符号的第一个比特。此时，反序列化器被称为已实现"符号锁定"。

COM 符号用于实现符号锁定，如下：
- 在链路训练期间，当链路首次建立或需要重新训练时，传输 TS1 和 TS2 有序集。
- 当发送 FTS 有序集以通知接收器将链路状态从 L0s 更改为 L0 时。

### 接收时钟补偿逻辑

#### 背景

我们之前观察到，链路两端的发送器和接收器使用的时钟不需要具有完全相同的频率。每当链路不使用公共参考时钟时，情况就会如此，并引入一个问题，即其中一个比另一个运行稍快。唯一的要求是两个时钟都必须在所需频率的 +/- 300 ppm（百万分之一）范围内。由于在最坏的情况下，一个可能是 +300 ppm，另一个可能是 -300 ppm，它们之间最坏情况的分离可能是 600ppm。这种差异转化为每 1666 个时钟增益或损失一个符号时钟。一旦链路被训练，接收器中的接收时钟与链路另一端的发送器中的发送时钟相同（因为接收时钟是从比特流派生的）。

#### 弹性缓冲区的作用

为了补偿这种最坏情况的频率差异，在接收路径中内置了弹性缓冲区。接收到的符号使用时钟同步到其中，使用时钟同步出，使用接收器的本地时钟。弹性缓冲区通过添加或删除 SKP 符号来补偿频率差异。当 SKP 有序集到达时，监视弹性缓冲区状态的逻辑进行评估。如果本地时钟运行更快，符号被时钟同步出的速度比进入的速度快，因此缓冲区将接近下溢条件。逻辑通过在到达时向有序集追加额外的 SKP 符号来补偿这一点，以快速重新填充缓冲区。另一方面，如果恢复的时钟运行更快，缓冲区将接近溢出条件，逻辑将通过删除其中一个 SKP 符号来补偿，以快速排空缓冲区。这些操作将弥补符号到达和消耗速率的差异，并防止任何混淆或数据丢失。

发送器定期发送 SKIP 有序集就是为了这个目的。顾名思义，SKP 字符实际上是可丢弃的字符。删除或添加 SKP 符号可防止弹性缓冲区溢出或下溢，然后当符号被转发到下一层时，它们与其他所有控制字符一起被丢弃。因此，它们使用一点带宽，但根本不会影响数据包的流动。

尽管由于弹性缓冲区溢出或下溢而丢失符号是错误条件，但接收器检查此情况是可选的。如果它们这样做，并且发生这种情况，将向数据链路层指示接收器错误。

发送器每 1180 到 1538 个符号时间调度一次 SKIP 有序集传输。但是，如果发送器恰好在调度传输 SKIP 有序集的 1538 个符号时间边界开始最大大小的 TLP 传输，则 SKIP 有序集传输被推迟。接收器必须能够容忍 SKIP 有序集具有最大间隔，该间隔取决于设备支持的最大数据包有效载荷大小。SKIP 有序集之间的最大符号数（n）公式为：

```
n = 1538 + (最大数据包有效载荷大小 + 28)
```

方程中的数字 28 是 TLP 开销。它是与头部（16 字节）、可选 ECRC（4 字节）、LCRC（4 字节）、序列号（2 字节）和帧符号 STP 和 END（2 字节）相关联的最大符号数。

### 通道间偏斜

#### 通道间飞行时间变化

对于宽链路，通道之间的偏斜是一个无法避免且必须在接收器处补偿的问题。符号使用相同的发送时钟同时在所有通道上发送，但不能指望它们在接收器处精确同时到达。通道间偏斜的来源包括：
- 电气驱动器和接收器之间的差异
- 印刷线路板阻抗变化
- 走线长度不匹配

当携带数据包的串行比特流到达接收器时，必须去除此通道间偏斜以按正确顺序接收字节。此过程称为链路的去偏斜。

#### 有序集帮助去偏斜

有序集的唯一结构以及它们同时在所有通道上发送的事实使它们可用于检测通道之间的时间未对齐。规范没有定义执行此操作的方法，但在 Gen1 和 Gen2 中，接收器逻辑可以简单地查找每个通道上的 COM 字符。如果它没有同时出现在所有通道上，则延迟早到达的 COM，直到它们在所有通道上匹配。

#### 接收器通道间去偏斜能力

这可以通过调整输入信号上的模拟延迟线来完成。或者，可以在弹性缓冲区之后完成，这具有到达时间差异现在被接收器的本地时钟数字化为符号时间的优势。如果一个通道的输入在时钟边沿上，而另一个没有，早到达的 COM 可以简单地延迟适当数量的符号时钟，以与晚到达的 COM 对齐。接收器处最大允许偏斜是时钟周期的倍数这一事实表明，规范编写者可能想到了这样的实现。

## 物理层错误处理

### 概述

物理层负责检测接收数据中的错误。如果检测到错误，将报告给数据链路层。数据链路层负责错误恢复，如第 10 章所述。

### 数据链路层对接收器错误的响应

当物理层检测到接收错误时，它会向数据链路层报告。数据链路层可以采取适当的措施，例如请求重新训练链路或丢弃损坏的数据包。

## 主动状态电源管理

主动状态电源管理（ASPM）允许链路在不活动时进入低功耗状态。物理层支持通过发送适当的电气空闲有序集来进入和退出这些状态。

## 链路训练和初始化

链路训练和初始化是建立和配置 PCIe 链路的过程。这包括位锁定、符号锁定、链路宽度协商和速度协商。有关详细信息，请参阅第 14 章"链路初始化和训练"。

## 总结

本章详细描述了 PCIe 物理层逻辑子模块的工作原理，特别是针对 Gen1 和 Gen2 操作。关键概念包括：

1. **字节分段和反分段**：将数据分布在多个通道上以提高带宽
2. **加扰**：减少 EMI 和串扰
3. **8b/10b 编码**：提供时钟恢复、直流平衡和错误检测
4. **有序集**：用于链路管理和维护的特殊控制序列
5. **弹性缓冲区**：补偿发送器和接收器之间的时钟差异
6. **通道去偏斜**：对齐多通道链路中的数据

这些机制共同确保 PCIe 链路能够可靠地以高速传输数据，同时保持信号完整性和电源效率。

### 11.16.1 RK3588 PCIe PHY物理层实现

#### PHY寄存器配置

```c
// 来自 drivers/phy/rockchip/phy-rockchip-pcie.c
#define PHY_CFG_DATA_SHIFT    7
#define PHY_CFG_ADDR_SHIFT    1
#define PHY_CFG_DATA_MASK     0xf
#define PHY_CFG_ADDR_MASK     0x3f
#define PHY_CFG_PLL_LOCK      0x10    // PLL锁定配置
#define PHY_CFG_CLK_TEST      0x10    // 时钟测试
#define PHY_CFG_CLK_SCC       0x12    // 时钟配置
#define PHY_CFG_SEPE_RATE     BIT(3)  // 信号速率
#define PHY_CFG_PLL_100M      BIT(3)  // 100M PLL
#define PHY_PLL_LOCKED        BIT(9)  // PLL锁定状态
```

**与书中11.2节的对应**：
- `PHY_CFG_SEPE_RATE` 对应书中发送器去加重（De-emphasis）配置
- `PHY_CFG_PLL_100M` 对应书中13.5节的参考时钟要求

#### PHY配置读写函数

```c
// PHY配置写入（对应书中物理层寄存器访问）
static inline void phy_wr_cfg(struct rockchip_pcie_phy *rk_phy,
                              u32 addr, u32 data)
{
    regmap_write(rk_phy->reg_base, rk_phy->phy_data->pcie_conf,
                 HIWORD_UPDATE(data,
                               PHY_CFG_DATA_MASK,
                               PHY_CFG_DATA_SHIFT) |
                 HIWORD_UPDATE(addr,
                               PHY_CFG_ADDR_MASK,
                               PHY_CFG_ADDR_SHIFT));
    udelay(1);
    // 写使能序列
    regmap_write(rk_phy->reg_base, rk_phy->phy_data->pcie_conf,
                 HIWORD_UPDATE(PHY_CFG_WR_ENABLE,
                               PHY_CFG_WR_MASK,
                               PHY_CFG_WR_SHIFT));
    udelay(1);
    regmap_write(rk_phy->reg_base, rk_phy->phy_data->pcie_conf,
                 HIWORD_UPDATE(PHY_CFG_WR_DISABLE,
                               PHY_CFG_WR_MASK,
                               PHY_CFG_WR_SHIFT));
}

// PHY配置读取
static inline u32 phy_rd_cfg(struct rockchip_pcie_phy *rk_phy,
                             u32 addr)
{
    u32 val;
    regmap_write(rk_phy->reg_base, rk_phy->phy_data->pcie_conf,
                 HIWORD_UPDATE(addr,
                               PHY_CFG_RD_MASK,
                               PHY_CFG_ADDR_SHIFT));
    regmap_read(rk_phy->reg_base,
                rk_phy->phy_data->pcie_status,
                &val);
    return val;
}
```

### 11.16.2 PLL锁定与链路训练

RK3588 PHY的PLL锁定过程对应书中14.4节的Detect状态：

```c
// 来自 phy-rockchip-pcie.c
static int rockchip_pcie_phy_power_on(struct phy *phy)
{
    // ...
    // 释放PHY复位（对应书中退出复位状态）
    err = reset_control_deassert(rk_phy->phy_rst);
    
    // 等待PLL锁定（对应书中时钟恢复）
    timeout = jiffies + msecs_to_jiffies(1000);
    while (time_before(jiffies, timeout)) {
        regmap_read(rk_phy->reg_base,
                    rk_phy->phy_data->pcie_status,
                    &status);
        if (status & PHY_PLL_LOCKED) {
            dev_dbg(&phy->dev, "pll locked!\n");
            err = 0;
            break;
        }
        msleep(20);
    }
    
    // 配置时钟测试（对应书中11.2节发送逻辑）
    phy_wr_cfg(rk_phy, PHY_CFG_CLK_TEST, PHY_CFG_SEPE_RATE);
    phy_wr_cfg(rk_phy, PHY_CFG_CLK_SCC, PHY_CFG_PLL_100M);
    
    // 等待PLL输出使能
    while (time_before(jiffies, timeout)) {
        regmap_read(rk_phy->reg_base,
                    rk_phy->phy_data->pcie_status,
                    &status);
        if (!(status & PHY_PLL_OUTPUT)) {
            dev_dbg(&phy->dev, "pll output enable done!\n");
            err = 0;
            break;
        }
        msleep(20);
    }
}
```

**与书中内容的对应**：

1. **PLL锁定**（对应14.4节Detect状态）：
   - 等待`PHY_PLL_LOCKED`对应书中时钟恢复完成
   - 超时处理对应书中训练超时机制

2. **时钟配置**（对应13.5节）：
   - `PHY_CFG_PLL_100M`对应书中100MHz参考时钟要求
   - `PHY_CFG_SEPE_RATE`对应书中发送速率配置

- `PHY_LANE_IDLE_OFF` 对应书中L0s状态的电气空闲
- `reset_control_assert` 对应书中L2/L3状态的时钟关闭

### 11.16.6 实际应用建议

**PHY调试技巧**：
1. 检查PLL锁定状态（`PHY_PLL_LOCKED`）
2. 验证通道检测状态（`PHY_LANE_RX_DET`）
3. 监控电源状态转换

**性能优化**：
- 合理配置去加重（`PHY_CFG_SEPE_RATE`）
- 优化PLL锁定等待时间
- 配置适当的电源管理策略
